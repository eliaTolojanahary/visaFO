# Sprint 3 : Finalisation et Scan (Scan Terminé)

## Contexte

L’opérateur a saisi la demande (Sprint 1) et a éventuellement géré un duplicata (Sprint 2).  
Il possède maintenant les documents physiques du demandeur.  
Il doit les numériser et les rattacher à chaque pièce justificative attendue.

Une fois toutes les pièces scannées, l’opérateur verrouille le dossier : plus aucune modification possible sur la demande, le demandeur ou le passeport.  
Le statut de la demande passe alors de `CRÉÉ` ou `En cours de traitement` à **`SCAN TERMINÉ`**.

## Objectif

* Ajouter la gestion des fichiers scannés (upload) par pièce justificative.
* Contrôler la complétude : bouton **« Scan Terminé »** actif uniquement quand **toutes** les pièces attendues (cochées dans `demande_piece`) ont un fichier associé.
* Verrouillage définitif des données après clic sur « Scan Terminé ».
* Nouveau statut `SCAN TERMINÉ` dans la table `statut_demande`.

## 2. Back-end - Elia  
**Branche :** `sprint/3/feature/scan-backend`

### Nouvelles classes (DAO, Repository, Service, Controller)

#### DAO
- `PieceFournieDAO`
  - `create(PieceFournie p)`
  - `findByDemandeAndPieceRef(demandeId, pieceRefId)`
  - `findAllByDemande(demandeId)`
  - `deleteByDemandeAndPieceRef(demandeId, pieceRefId)`

#### Repository
- `PieceFournieRepository` (wrapper appelant le DAO)
  - `sauvegarder(piece)`
  - `recuperer(demandeId, pieceRefId)`
  - `listerParDemande(demandeId)`
  - `supprimer(demandeId, pieceRefId)`

#### Service
- `ScanService`
  - `uploadPiece(demandeId, pieceRefId, Part fichier)` : sauvegarde le fichier sur disque et enregistre en base. Remplace l’existant. Vérifie taille max 10 Mo et types autorisés (jpeg, png, pdf). Lance `DemandeVerrouilleeException` si la demande est verrouillée.
  - `isDemandeComplete(demandeId)` : retourne `true` si pour chaque ligne de `demande_piece` avec `cochee = true` il existe une `piece_fournie` correspondante.
  - `verrouillerDemande(demandeId)` : vérifie la complétude, passe `verrouille = true`, change le statut de la demande à `SCAN TERMINÉ`.

- `DemandeService` (modification)
  - Toutes les méthodes de mise à jour (`updateDemande`, `updateDemandeur`, `updatePasseport`) doivent vérifier `demande.verrouille == false` avant exécution. Sinon lancer `DemandeVerrouilleeException`.

#### Contrôleurs MVC (pas REST)

Classe `ScanController` :

- `@GetMapping("/demande/{id}/scan")`
  - Affiche la page `scanDemande.jsp`
  - Modèle : `demande`, `listePiecesAttendues` (avec pour chacune `pieceFournie` associée ou null), `demandeComplete` (booléen)
  - Si la demande est déjà verrouillée, affiche un message bloquant et désactive les formulaires.

- `@PostMapping("/demande/{demandeId}/piece/{pieceRefId}/upload")`
  - Reçoit `MultipartFile` ou `Part`, appelle `scanService.uploadPiece()`
  - Redirige vers `GET /demande/{demandeId}/scan` avec message flash de succès ou d’erreur.

- `@PostMapping("/demande/{demandeId}/verrouiller")`
  - Appelle `scanService.verrouillerDemande()`
  - Redirige vers `/suivi?ref={refDemande}` (page de suivi du dossier du Sprint 2) avec message flash de confirmation.

- `@GetMapping("/demande/{demandeId}/piece/{pieceRefId}/download")`
  - Télécharge le fichier scanné (définit `Content-Type`, `Content-Disposition`).

### Contrôleur complémentaire 
Classe `SuiviController` (ou ajout dans `DossierController`) :
- `@GetMapping("/suivi")` avec paramètre `ref` (ref_demande) – déjà existant au Sprint 2
- À enrichir pour fournir à la JSP la liste des `piece_fournie`

---
### Règles métier à implémenter

* Taille max fichier : 10 Mo
* Types MIME autorisés : `image/jpeg`, `image/png`, `application/pdf`
* Une pièce déjà scannée peut être remplacée (suppression physique de l’ancien fichier)
* Le verrouillage est irréversible
* Si aucune pièce attendue (`demande_piece` avec `cochee = true` est vide), `isDemandeComplete()` retourne `true`

## 4. Routes complètes (contrôleurs MVC)

### ScanController

| Route | Méthode | Description |
|-------|---------|-------------|
| `/demande/{id}/scan` | GET | Affiche la page de gestion des scans |
| `/demande/{demandeId}/piece/{pieceRefId}/upload` | POST | Reçoit le fichier scanné pour une pièce |
| `/demande/{demandeId}/verrouiller` | POST | Verrouille la demande (si complète) |
| `/demande/{demandeId}/piece/{pieceRefId}/download` | GET | Télécharge le fichier scanné |

### SuiviController (extension)

| Route | Méthode | Description |
|-------|---------|-------------|
| `/suivi` | GET | Affiche le suivi du dossier (ref demande en param). Ajoute `piecesScannees` au modèle. |

### DemandeController (modification)

Les routes de modification existantes (`/visa/form/update`, etc.) doivent maintenant vérifier le flag `verrouille`. En cas de tentative sur une demande verrouillée, rediriger vers une page d’erreur ou afficher un message.
