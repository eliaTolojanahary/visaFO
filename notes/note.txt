Projet Mr Naina :

Contexte : etranger => demande de visa . Visa => transformable et non transformable 
- Visa transformable : 
    - Permet de changer de statut (ex : étudiant => salarié)
    - Nécessite une demande auprès de la préfecture
    - Doit être fait avant l'expiration du visa actuel  

- Visa non transformable : visa long sejour (ex : visa de court séjour, visa de transit)
    - Ne permet pas de changer de statut
    - Doit quitter le territoire à l'expiration du visa
    - Peut nécessiter une nouvelle demande de visa pour revenir en France

Visa transformable :
   - Exemples : visa étudiant, visa de travail
   - Processus : 
     1. Faire une demande de changement de statut auprès de la préfecture
     2. Fournir les documents nécessaires (contrat de travail, preuve d'inscription à l'université, etc.)
     3. Attendre la décision de la préfecture

Contrainte de l'app : 
   - Cas reel : base de donne des visa 
   - Cas ici :  base fictif de visa dans l'app (saisi des donne directement dans l'app)

- Fonctionnalités de l'app :
    - Saisie des données de visa (type, durée, statut actuel, etc.)
    - Demande de changement de statut pour les visas transformables 
        => formulaire de demande : fiche de renseignement, etat civil, visa actuel visa demander
        => information specifique a rechercher (document a fournir, delai de traitement, etc.)
        => type de visa traiter travailleur, etudiant 
    - Securite : transformation de visa => envoi de demande avant le changement de statut


Demande de visa :
Type : 
    - nouveau titre,
    - duplicata (carte resident),
    - transfert de visa (pour le visa uniquement) ,
    - renouvellement

NB : donne peuvent changer en fonction du type de demande (ex : num passeport,  duplicata => perte ou vol, transfert => changement de statut, etc.)
 duplicata (carte resident) et transfert de visa (pour le visa uniquement) non soumis dans le doone existanxte 
 si donne anterieur existe deja => mise a jour de la demande (ex : renouvellement)

Status : chaque demande devrait etre enregistre 
    - en cours de traitement
    - accepté
    - refusé

Processus : 
    Finalite carte resident + visa (meme date fin de validite)
